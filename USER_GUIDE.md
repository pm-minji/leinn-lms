## 🎯 시스템 소개

### LEINN LMS란?

LEINN LMS는 **팀 기반 창업 교육**을 위한 학습 관리 시스템입니다. 몬드라곤 대학의 LEINN 프로그램을 기반으로 한 "Learning-by-Doing" 교육 모델을 한국 교육 환경에 맞게 구현했습니다.

### 핵심 개념

#### 🏆 팀 기반 학습
- 학습의 기본 단위는 **팀**입니다
- 각 팀은 여러 학습자로 구성됩니다
- 팀원들은 함께 프로젝트를 수행하며 학습합니다

#### 📝 리플렉션 (성찰)
- 학습자가 주간 단위로 작성하는 학습 일지입니다
- 학습 진행 상황, 어려움, 인사이트를 기록합니다
- AI가 자동으로 분석하여 코치에게 인사이트를 제공합니다

#### 🤖 AI 코칭 지원
- Google Gemini AI가 리플렉션을 분석합니다
- 요약, 리스크 식별, 액션 아이템을 자동 생성합니다
- 코치만 AI 분석 결과를 볼 수 있습니다

#### 👨‍🏫 코치 피드백
- 코치가 AI 분석을 참고하여 개인화된 피드백을 제공합니다
- 학습자의 성장을 돕는 구체적인 조언을 포함합니다

### 사용자 역할

| 역할 | 설명 | 주요 기능 |
|------|------|-----------|
| **학습자** | 팀에 소속되어 학습하는 학생 | 리플렉션 작성, 피드백 확인 |
| **코치** | 학습자를 멘토링하는 강사 | 피드백 작성, 코칭 로그 관리 |
| **관리자** | 시스템을 운영하는 관리자 | 팀 관리, 사용자 관리, 시스템 설정 |

## 🚀 시작하기

### 1단계: 로그인

1. 웹사이트에 접속합니다
2. **"Google로 로그인"** 버튼을 클릭합니다
3. Google 계정으로 인증을 완료합니다

> 💡 **팁**: 처음 로그인하면 자동으로 **학습자** 역할이 부여됩니다. 코치나 관리자 권한이 필요한 경우 시스템 관리자에게 문의하세요.

### 2단계: 프로필 확인

로그인 후 우측 상단의 프로필 아이콘을 클릭하여 다음을 확인하세요:
- 이름과 이메일이 올바른지 확인
- 현재 역할 확인
- 소속 팀 정보 확인 (학습자인 경우)

### 3단계: 대시보드 둘러보기

역할에 따라 다른 대시보드가 표시됩니다:
- **학습자**: 리플렉션 현황, 최근 피드백
- **코치**: 담당 팀 현황, 피드백 대기 목록
- **관리자**: 전체 시스템 현황, 관리 메뉴

## 👨‍🎓 학습자 가이드

### 대시보드 이해하기

학습자 대시보드에서 확인할 수 있는 정보:

#### 📊 리플렉션 통계
- 이번 주 제출 여부
- 총 제출한 리플렉션 수
- 피드백 완료율

#### 📋 최근 리플렉션
- 최근 제출한 리플렉션 목록
- 각 리플렉션의 상태 (제출됨, AI 분석 완료, 코치 피드백 완료)

### 리플렉션 작성하기

#### 1단계: 새 리플렉션 시작

1. 좌측 메뉴에서 **"리플렉션"** 클릭
2. **"새 리플렉션 작성"** 버튼 클릭
3. 리플렉션 작성 페이지로 이동

#### 2단계: 효과적인 리플렉션 작성

리플렉션은 최소 **100자 이상** 작성해야 합니다. 다음 가이드를 참고하세요:

##### ✅ 좋은 리플렉션의 예시

```
이번 주 팀 프로젝트에서 마케팅 전략을 담당했습니다. 

**진행 상황:**
- 타겟 고객 분석 완료 (20-30대 직장인)
- 경쟁사 분석 3개 업체 완료
- SNS 마케팅 계획 초안 작성

**어려웠던 점:**
- 정확한 시장 규모 파악이 어려웠습니다
- 팀원들과 의견 조율에 시간이 많이 걸렸습니다

**배운 점:**
- 데이터 기반 의사결정의 중요성을 깨달았습니다
- 효과적인 커뮤니케이션 방법을 배웠습니다

**다음 주 계획:**
- 시장 조사 보완
- 프로토타입 제작 시작
```

##### ❌ 피해야 할 리플렉션

```
이번 주도 열심히 했습니다. 다음 주도 열심히 하겠습니다.
```

#### 3단계: 리플렉션 제출

1. 내용을 충분히 작성했는지 확인
2. **"제출하기"** 버튼 클릭
3. 제출 확인 메시지 확인

### 리플렉션 상태 이해하기

| 상태 | 설명 | 다음 단계 |
|------|------|-----------|
| **제출됨** | 리플렉션이 성공적으로 제출됨 | AI 분석 대기 중 |
| **AI 분석 완료** | AI가 분석을 완료함 | 코치 피드백 대기 중 |
| **코치 피드백 완료** | 코치가 피드백을 작성함 | 피드백 확인 가능 |

### 피드백 확인하기

#### 코치 피드백 보기

1. **"리플렉션"** 메뉴 클릭
2. 피드백을 확인하고 싶은 리플렉션 선택
3. 하단의 **"코치 피드백"** 섹션 확인

#### 피드백 활용하기

- 코치의 조언을 다음 주 학습에 반영하세요
- 궁금한 점이 있으면 코치에게 직접 문의하세요
- 피드백을 바탕으로 개선 계획을 세우세요

### 과거 리플렉션 관리

#### 리플렉션 목록 보기

- 모든 리플렉션이 **최신순**으로 정렬됩니다
- 각 리플렉션의 제목, 날짜, 상태를 한눈에 확인할 수 있습니다

#### 검색 및 필터링

- 특정 기간의 리플렉션을 찾고 싶을 때 활용하세요
- 상태별로 필터링하여 피드백이 완료된 리플렉션만 볼 수 있습니다

## 👨‍🏫 코치 가이드

### 코치 대시보드 이해하기

코치 대시보드에서 확인할 수 있는 정보:

#### 📈 담당 팀 현황
- 각 팀의 리플렉션 제출률
- 피드백 완료 현황
- 팀별 학습자 수

#### ⏰ 피드백 대기 목록
- AI 분석이 완료되어 피드백이 필요한 리플렉션
- 우선순위별 정렬

#### 📅 후속 일정 알림
- 예정된 코칭 세션
- 후속 조치가 필요한 항목

### 팀 관리하기

#### 담당 팀 확인

1. **"팀 관리"** 메뉴 클릭
2. 담당하는 모든 팀 목록 확인
3. 각 팀을 클릭하여 상세 정보 확인

#### 팀 상세 정보

각 팀 페이지에서 확인할 수 있는 정보:
- 팀 소속 학습자 목록
- 각 학습자의 최근 리플렉션 제출 현황
- 팀 전체 활동 요약

### AI 피드백 활용하기

#### AI 분석 결과 이해

AI가 제공하는 정보:

1. **📋 요약 (Summary)**
   - 리플렉션의 핵심 내용 요약
   - 주요 키워드와 테마 추출

2. **⚠️ 리스크 (Risks)**
   - 학습자가 직면한 잠재적 문제
   - 주의 깊게 관찰해야 할 신호

3. **🎯 액션 아이템 (Actions)**
   - 구체적인 개선 제안
   - 다음 단계 권장사항

#### AI 피드백 활용 팁

- AI 분석을 **참고 자료**로 활용하세요
- 학습자의 개별 상황을 고려하여 해석하세요
- AI가 놓칠 수 있는 감정적, 맥락적 요소를 보완하세요

### 효과적인 코치 피드백 작성하기

#### 피드백 작성 과정

1. **리플렉션 검토**
   - 학습자의 리플렉션을 꼼꼼히 읽어보세요
   - AI 분석 결과를 참고하세요

2. **피드백 작성**
   - 구체적이고 실행 가능한 조언을 제공하세요
   - 긍정적인 부분과 개선점을 균형있게 언급하세요

3. **피드백 제출**
   - 작성 완료 후 **"피드백 제출"** 버튼 클릭

#### 좋은 피드백의 구조

```
**잘한 점:**
- 구체적인 데이터를 활용한 분석이 인상적입니다
- 팀원들과의 협업 과정을 잘 기록했네요

**개선 제안:**
- 시장 조사 방법론을 더 체계화해보세요
- 다음 주에는 고객 인터뷰를 진행해보는 것을 추천합니다

**질문:**
- 현재 가장 큰 어려움은 무엇인가요?
- 팀 내 역할 분담은 어떻게 되어 있나요?

**다음 주 포커스:**
- 고객 검증에 집중해보세요
- 정기적인 팀 회의 시간을 정해보세요
```

### 코칭 로그 관리하기

#### 코칭 로그란?

코칭 로그는 학습자나 팀과의 **직접적인 코칭 세션**을 기록하는 도구입니다.

#### 코칭 로그 작성하기

1. **"코칭 로그"** 메뉴 클릭
2. **"새 코칭 로그 작성"** 버튼 클릭
3. 다음 정보를 입력:

| 필드 | 설명 | 예시 |
|------|------|------|
| **세션 제목** | 코칭 세션의 주제 | "마케팅 전략 검토" |
| **날짜** | 코칭 세션 날짜 | 2024-01-15 |
| **유형** | 1:1, 팀, 주간 중 선택 | 1:1 |
| **대상** | 학습자 또는 팀 선택 | 김철수 |
| **메모** | 세션 내용 기록 | 상세한 논의 내용 |
| **다음 액션** | 후속 조치 사항 | 시장 조사 보완 |
| **후속 일정** | 다음 만남 예정일 | 2024-01-22 |

#### 코칭 로그 활용하기

- **연속성 유지**: 이전 세션 내용을 참고하여 일관된 코칭 제공
- **진행 상황 추적**: 학습자의 성장 과정을 체계적으로 기록
- **팀 코칭**: 팀 전체의 역학 관계와 협업 상황 파악

## 👨‍💼 관리자 가이드

### 관리자 대시보드

관리자는 시스템의 모든 기능에 접근할 수 있으며, 다음과 같은 관리 업무를 수행합니다:

#### 📊 시스템 현황
- 전체 사용자 수 (학습자, 코치)
- 활성 팀 수
- 리플렉션 제출 현황
- AI 분석 처리 상태

### 팀 관리

#### 새 팀 생성하기

1. **"팀 관리"** 메뉴 클릭
2. **"새 팀 생성"** 버튼 클릭
3. 팀 정보 입력:
   - **팀 이름**: 고유한 팀 이름
   - **설명**: 팀의 목적과 특성 (선택사항)
   - **활성 상태**: 활성/비활성 선택

#### 팀 정보 수정하기

1. 팀 목록에서 수정할 팀 선택
2. **"편집"** 버튼 클릭
3. 정보 수정 후 저장

#### 팀 비활성화하기

- 더 이상 사용하지 않는 팀은 **비활성** 상태로 변경
- 비활성 팀의 데이터는 보존되지만 새로운 활동은 제한됨

### 사용자 관리

#### 사용자 역할 변경하기

1. **"사용자 관리"** 메뉴 클릭
2. 역할을 변경할 사용자 선택
3. **"역할 변경"** 버튼 클릭
4. 새로운 역할 선택:
   - **학습자**: 기본 역할
   - **코치**: 팀 멘토링 권한
   - **관리자**: 전체 시스템 관리 권한

#### 학습자 팀 배정하기

1. **"학습자 관리"** 메뉴 클릭
2. 팀을 배정할 학습자 선택
3. **"팀 배정"** 버튼 클릭
4. 배정할 팀 선택

> ⚠️ **주의**: 한 학습자는 하나의 팀에만 소속될 수 있습니다.

#### 코치 팀 할당하기

1. **"코치 관리"** 메뉴 클릭
2. 팀을 할당할 코치 선택
3. **"팀 할당"** 버튼 클릭
4. 할당할 팀들을 선택 (복수 선택 가능)

> 💡 **팁**: 한 코치는 여러 팀을 담당할 수 있습니다.

### AI 프롬프트 관리

#### AI 프롬프트란?

AI 프롬프트는 Google Gemini가 리플렉션을 분석할 때 사용하는 **지시사항**입니다. 프롬프트를 조정하여 AI 분석의 방향과 품질을 개선할 수 있습니다.

#### 프롬프트 관리하기

1. **"AI 프롬프트 관리"** 메뉴 클릭
2. 현재 활성 프롬프트와 전체 프롬프트 목록 확인

#### 새 프롬프트 생성하기

1. **"새 프롬프트 생성"** 버튼 클릭
2. 프롬프트 정보 입력:
   - **이름**: 프롬프트 식별용 이름
   - **설명**: 프롬프트의 목적과 특징
   - **프롬프트 내용**: 실제 AI에게 전달할 지시사항

#### 프롬프트 변수 활용하기

프롬프트에서 사용할 수 있는 변수:

| 변수 | 설명 | 예시 |
|------|------|------|
| `{reflection_content}` | 리플렉션 내용 | 학습자가 작성한 전체 텍스트 |
| `{learner_name}` | 학습자 이름 | 김철수 |
| `{team_name}` | 팀 이름 | Alpha Team |
| `{week_start}` | 주차 시작일 | 2024-01-15 |

#### 프롬프트 예시

```
당신은 창업 교육 전문 코치입니다. 다음 학습자의 리플렉션을 분석해주세요.

학습자: {learner_name}
팀: {team_name}
주차: {week_start}
```

## ❓ 자주 묻는 질문 (FAQ)

### 일반적인 질문

#### Q: 리플렉션은 얼마나 자주 작성해야 하나요?
A: **주 1회** 작성을 권장합니다. 매주 같은 요일에 작성하는 습관을 만들어보세요.

#### Q: 리플렉션 작성에 시간 제한이 있나요?
A: 시간 제한은 없습니다. 충분히 생각하고 작성하세요. 보통 10-15분 정도 소요됩니다.

#### Q: AI 피드백을 학습자도 볼 수 있나요?
A: 아니요. AI 피드백은 **코치만** 볼 수 있습니다. 학습자는 코치가 작성한 최종 피드백만 확인할 수 있습니다.

#### Q: 팀을 변경할 수 있나요?
A: 팀 변경은 관리자만 가능합니다. 필요한 경우 관리자에게 문의하세요.

### 기술적인 질문

#### Q: 어떤 브라우저를 사용해야 하나요?
A: Chrome, Firefox, Safari, Edge 등 최신 브라우저를 권장합니다.

#### Q: 모바일에서도 사용할 수 있나요?
A: 네, 반응형 디자인으로 모바일, 태블릿에서도 사용 가능합니다.

#### Q: 데이터는 안전하게 보관되나요?
A: 네, 모든 데이터는 암호화되어 안전하게 저장됩니다. Google OAuth를 통한 보안 인증을 사용합니다.

### 역할별 질문

#### 학습자

**Q: 리플렉션을 제출한 후 수정할 수 있나요?**
A: 제출 후에는 수정이 불가능합니다. 제출 전에 충분히 검토해주세요.

**Q: 코치 피드백은 언제 받을 수 있나요?**
A: 보통 리플렉션 제출 후 2-3일 내에 피드백을 받을 수 있습니다.

#### 코치

**Q: 여러 팀을 동시에 담당할 수 있나요?**
A: 네, 관리자가 설정하면 여러 팀을 담당할 수 있습니다.

**Q: AI 분석 결과가 부정확하다면 어떻게 하나요?**
A: AI 분석은 참고용입니다. 코치의 전문적 판단을 우선하여 피드백을 작성하세요.

#### 관리자

**Q: 사용자 데이터를 백업할 수 있나요?**
A: 네, Supabase 대시보드에서 데이터 백업이 가능합니다.

**Q: AI 프롬프트 변경이 기존 데이터에 영향을 주나요?**
A: 아니요, 프롬프트 변경은 이후 제출되는 리플렉션부터만 적용됩니다.

## 🔧 문제 해결

### 로그인 문제

#### 문제: Google 로그인이 안 됩니다
**해결 방법:**
1. 브라우저 쿠키와 캐시를 삭제해보세요
2. 시크릿/프라이빗 모드에서 시도해보세요
3. 다른 브라우저에서 시도해보세요
4. Google 계정이 정상적으로 작동하는지 확인하세요

#### 문제: 로그인 후 권한 오류가 발생합니다
**해결 방법:**
1. 관리자에게 계정 권한 확인을 요청하세요
2. 올바른 Google 계정으로 로그인했는지 확인하세요
3. 브라우저를 새로고침해보세요

### 리플렉션 관련 문제

#### 문제: 리플렉션 제출이 안 됩니다
**해결 방법:**
1. 리플렉션 내용이 100자 이상인지 확인하세요
2. 인터넷 연결 상태를 확인하세요
3. 페이지를 새로고침하고 다시 시도하세요
4. 브라우저 콘솔에서 에러 메시지를 확인하세요

#### 문제: AI 분석이 오래 걸립니다
**해결 방법:**
1. AI 분석은 보통 1-2분 정도 소요됩니다
2. 5분 이상 걸린다면 페이지를 새로고침해보세요
3. 시스템 상태를 관리자에게 문의하세요

### 피드백 관련 문제

#### 문제: 코치 피드백이 보이지 않습니다
**해결 방법:**
1. 리플렉션 상태가 "코치 피드백 완료"인지 확인하세요
2. 페이지를 새로고침해보세요
3. 코치에게 피드백 작성 여부를 확인하세요

#### 문제: 피드백 작성이 안 됩니다 (코치)
**해결 방법:**
1. 담당 팀의 리플렉션인지 확인하세요
2. AI 분석이 완료된 리플렉션인지 확인하세요
3. 피드백 내용이 충분히 작성되었는지 확인하세요

### 성능 문제

#### 문제: 페이지 로딩이 느립니다
**해결 방법:**
1. 인터넷 연결 속도를 확인하세요
2. 브라우저 캐시를 삭제해보세요
3. 다른 기기에서 접속해보세요
4. 시스템 상태를 관리자에게 문의하세요

#### 문제: 모바일에서 화면이 깨집니다
**해결 방법:**
1. 브라우저를 최신 버전으로 업데이트하세요
2. 화면을 세로 모드로 회전해보세요
3. 브라우저 줌 레벨을 100%로 설정하세요

### 긴급 상황 대응

#### 시스템 전체 장애
1. **즉시 관리자에게 연락**하세요
2. 작업 중이던 내용을 **별도로 백업**하세요
3. **공지사항**을 확인하여 복구 예정 시간을 파악하세요

#### 데이터 손실 의심
1. **즉시 작업을 중단**하세요
2. **관리자에게 긴급 연락**하세요
3. 문제 발생 시점과 상황을 **상세히 기록**하세요

### 지원 요청하기

문제가 해결되지 않는 경우:

1. **관리자 연락처**로 문의
2. 다음 정보를 포함하여 문의:
   - 사용자 이름과 역할
   - 문제 발생 시간
   - 구체적인 증상
   - 시도해본 해결 방법
   - 브라우저 종류와 버전
   - 에러 메시지 (있는 경우)

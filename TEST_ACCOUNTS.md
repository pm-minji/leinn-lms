# 🧪 테스트 계정 설정 가이드

## 📋 설정 순서

### 1️⃣ 계정 가입
먼저 https://leinn-lms.vercel.app 에서 다음 이메일로 가입하세요:

**관리자 계정:**
- 이메일: `skhansky@gmail.com`
- 비밀번호: (본인이 설정)

**테스트 계정들:**
- 관리자: `admin@test.com` / 비밀번호: `Test1234!`
- 코치: `coach@test.com` / 비밀번호: `Test1234!`
- 학습자: `learner@test.com` / 비밀번호: `Test1234!`

### 2️⃣ Supabase에서 역할 설정

1. Supabase 대시보드 접속: https://supabase.com/dashboard
2. 프로젝트 선택
3. **SQL Editor** 메뉴 클릭
4. `supabase/setup_test_accounts.sql` 파일 내용 복사
5. SQL Editor에 붙여넣기
6. **Run** 버튼 클릭

### 3️⃣ 확인

다시 로그인하면 각 계정의 역할에 맞는 대시보드가 표시됩니다:

- **관리자**: 사용자/팀 관리, AI 프롬프트 관리
- **코치**: 리플렉션 검토, 코칭 로그 작성
- **학습자**: 리플렉션 작성, 팀 정보 확인

## 🎯 테스트 시나리오

### 학습자 계정 (learner@test.com)
1. 로그인
2. "새 리플렉션 작성" 클릭
3. 제목, 날짜, 내용 작성 (최소 100자)
4. 제출 후 AI 분석 확인

### 코치 계정 (coach@test.com)
1. 로그인
2. 리플렉션 목록에서 학습자 리플렉션 확인
3. 피드백 작성
4. 코칭 로그 작성

### 관리자 계정 (admin@test.com 또는 skhansky@gmail.com)
1. 로그인
2. 사용자 관리 - 역할 변경 테스트
3. 팀 생성 및 멤버 할당
4. AI 프롬프트 템플릿 수정

## 🔧 문제 해결

**로그인 후 역할이 반영되지 않는 경우:**
1. 로그아웃
2. 브라우저 캐시 삭제
3. 다시 로그인

**SQL 실행 오류:**
- 먼저 해당 이메일로 가입이 완료되었는지 확인
- 이메일 인증이 완료되었는지 확인
